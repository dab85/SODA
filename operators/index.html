Соглашение об именовании переменных и подпрограмм

Для повышения читаемости и самодокументируемости кода рекомендуется использовать следующие префиксы.

1. Функции, возвращающие значение (геттеры)

Функции, которые возвращают значение, должны начинаться с префикса Get.

Назначение Префикс Пример (рус) Пример (англ)
Получение значения Get ПолучитьОстаток() GetRemaining()
Получение имени Get ПолучитьИмяПользователя() GetUserName()
Получение даты Get ПолучитьТекущуюДату() GetCurrentDate()

2. Процедуры, устанавливающие значения (сеттеры)

Процедуры, которые устанавливают значения свойств или состояния, должны начинаться с префикса Set.

Назначение Префикс Пример (рус) Пример (англ)
Установка параметров Set УстановитьПараметры() SetParameters()
Установка статуса Set УстановитьСтатус() SetStatus()
Установка скидки Set УстановитьСкидку() SetDiscount()

3. Булевы функции и переменные (предикаты)

Функции и переменные, которые возвращают или хранят логическое значение (Истина/Ложь), должны начинаться с префикса Is.

Назначение Префикс Пример (рус) Пример (англ)
Проверка типа Is ЭтоДокумент() IsDocument()
Флаг доступности Is Доступно IsAvailable
Проверка заполненности Is Заполнено IsFilled

Сводная таблица

Префикс Назначение
Get Получить значение (функция)
Set Установить значение (процедура)
Is Логическое значение (Истина/Ложь)

Дополнительные префиксы для булевых значений

Префикс Значение Пример (англ)
Has Имеет что-то HasChildren
Can Может выполнить действие CanEdit
Contains Содержит элементы ContainsErrors
